Notable design decisions:

  • Left menu, header, and footer share the same markup for each page
  • Dynamic Graph code can handle different bar values simply by changing constants and markup.
  • Dynamic Graph code does not need to be changed if graph height changes.
  • If graph width changes, you only need to change 2 constants in the Dynamic Graph code.
  • Intuitive UX: after entering a number on the dynamic graph page, the textbox is cleared and refocused
  • Number textbox uses validation rather than input filtering. Input filtering creates too many special cases, such as allowing the user to refresh with F5 while focused on the textbox.
  • Compativle with Chrome 46.0.2490.86 m, Firefox 42.0, and Internet Explorer 11.0.9600.18097. (IE users must click "Allow blocked content.")
